# Turbolink 活动指标公式参考

## 基础指标

| 指标 | 数据来源 | 说明 |
|------|----------|------|
| UV（独立访客数） | 活动列表接口 `visitor_num` 字段 | 过滤条件：仅保留 UV > 阈值的活动 |
| 点击活动主按钮人数 | `/fbi/report/custom` type=5 | 激励养成活动使用 type=16（领养人数） |

## 率值指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **活动参与率** | 点击活动主按钮人数 / UV | 衡量活动对用户的吸引力 |
| **任务完成率** | 完成任务人数 / UV | 衡量用户参与深度 |
| **次留（次日留存率）** | 活动开始后第2天活跃用户数 / 活动当天用户数 | 来源：留存接口 `day_1` 字段 |
| **7日留（7日留存率）** | 活动开始后第8天活跃用户数 / 活动当天用户数 | 来源：留存接口 `day_7` 字段 |

## 金币指标（仅金币大派送活动）

| 指标 | 公式 | 说明 |
|------|------|------|
| **回收金币** | 用户投放金币总数 - 用户领取金币总数 | 按天累加，统计截止到指定日期 |
| **金币回收比例** | 回收金币 / 用户投放金币总数 | 反映平台从用户侧净回收的金币占比 |
| **投放金币人数** | `/fbi/report/custom` type=38 | 向金币池投放金币的独立用户数 |
| **投放金币率** | 投放金币人数 / UV | 衡量活动用户中参与投放的比例 |
| **领取金币人数** | `/fbi/report/custom` type=39 | 从金币池领取金币的独立用户数 |
| **领取金币率** | 领取金币人数 / UV | 衡量活动用户中参与领取的比例 |

### 金币数据来源

通过 `/admin/fission-stat/reward-detail` 接口获取每日金币池数据：
- `user_coin`：用户投放金币数（用户通过游戏投入的金币）
- `get_coin`：用户领取金币数（用户从池中领走的金币）

统计范围：活动开始至指定截止日期（`--coin-cutoff` 参数），包含截止日当天。

## 赛事竞猜活动特殊逻辑

赛事竞猜活动（`fission_mark = "vs"`）输出到独立的「赛事竞猜」Sheet，包含两级表头：

### 活动级指标（行1表头）

| 指标 | 说明 |
|------|------|
| UV | 活动独立访客数 |
| 完成任务人数 | `/fbi/report/custom` type=11 |
| 任务完成率 | 完成任务人数 / UV |
| 完成任务次数 | type=11 的 `num` 字段 |

### 每场比赛指标（行2表头）

| 指标 | 数据来源 | 说明 |
|------|----------|------|
| 竞猜时间 | `schedules[].vs_date` | 开赛时间，按升序排列 |
| A队投票人数 | `schedules[].votes[0].vote_user_num` | 投A队赢的独立用户数 |
| 平局投票人数 | `schedules[].votes[2].vote_user_num` | 投平局的独立用户数 |
| B队投票人数 | `schedules[].votes[1].vote_user_num` | 投B队赢的独立用户数 |
| 单次参加投票人数 | A队 + 平局 + B队投票人数 | 该场比赛总投票用户数 |
| 胜队 | `schedules[].win_type` + `win_team_id` | win_type=0: 未开奖(/)、=1: 某队胜、=2: 平局 |
| 领奖人数 | `schedules[].reward_num` | 该场比赛领取奖励的用户数 |
| 领奖率 | 领奖人数 / 胜队投票人数 | 胜队投票人数为0时显示/ |

### 赛事数据来源

通过 `/admin/fission-stat/reward-detail` 接口获取 `schedules` 数组：
- `teams[]`：队伍信息（id, name）
- `votes[]`：投票数据（win_type, win_team_id, vote_num, vote_user_num）
- `win_type`：0=未开奖, 1=某队胜, 2=平局
- `win_team_id`：胜队 ID（win_type=1 时有效）

## 激励养成活动特殊逻辑

激励养成活动（`fission_mark = "keep"`）有两行数据：
1. **领养人数**（type=16）：参与活动的用户数，计入"点击活动主按钮人数"
2. **养成人数**（type=30）：完成养成的用户数，单独一行展示

其他活动类型只有一行点击数据。
